## 📋 Notes

### Architecture — API-only + OpenClaw Interface
```
Discord (OpenClaw bot)
    ↓
openclaw-gateway (Tailscale only, port 18789)
    ├── GitHub API (readonly PAT) — commit/PR/pipeline tracking
    ├── ra-app:3500/v1/chat/completions — LLM via RustAssistant proxy
    ├── ra-app:3500/api/v1/search — semantic search over session memory
    └── ra-app:3500/api/v1/repos/:id/sync — trigger manual sync

ra-app:3500 (API only — no WebUI)
    ├── /v1/*           — OpenAI-compatible proxy (xAI default / Ollama optional)
    ├── /api/v1/*       — REST API (repos, docs, search, chat, audit)
    ├── /api/github/*   — webhook receiver
    ├── /healthz        — health check
    └── /metrics        — Prometheus metrics
```

### Architecture — LLM Provider Routing
```
OpenClaw / futures-app / any OpenAI-compatible client
    ↓
POST /v1/chat/completions (OpenAI standard wire format)
    ↓
ModelRouter::classify_prompt_async()
    ├── xAI (Grok) ← DEFAULT for all requests when Ollama is not available
    │
    ├── (when Ollama is enabled and reachable):
    │   ├── ScaffoldStub / TodoTagging / TreeSummary → Ollama (local)
    │   ├── RepoQuestion / SymbolExtraction         → Ollama (local, fast)
    │   └── ArchitecturalReason / CodeReview        → Grok (remote)
    │
    └── Unknown / force_remote / Ollama unreachable → Grok (remote, fallback)
```

### OpenClaw Session Flow
```
Pre-session:
  @openclaw brief  →  fetch recent commits + pipeline status + RA semantic search
                   →  Discord message: "Last session: X commits. Failing: Y. Focus: Z"

During session:
  Commit pushed → GitHub webhook → ra-app syncs repo → embeddings updated
  Pipeline run  → GitHub Actions status → OpenClaw notifies Discord on failure

Post-session:
  @openclaw wrap  →  fetch commits since last wrap + pipeline outcomes
                  →  write sessions/YYYY-MM-DD.md to workspace
                  →  POST to ra-app /api/v1/documents (index session notes)
                  →  Discord message: "Session summary + TODO list for next time"
```

### Pipeline Philosophy
- `todo.md` is the **single source of truth**.
- `todo-scaffold` is always safe to re-run — idempotent.
- `todo-work` never touches files without creating backups.
- `todo-work` runs `cargo check` after patching; rolls back on failure.
- **IDs are stable**: 8-char hex CRC32 of the raw list line.
- **`--auto-sync`**: skip the manual sync step entirely.
